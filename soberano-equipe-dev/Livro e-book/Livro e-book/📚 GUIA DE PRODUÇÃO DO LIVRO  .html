# Capítulo 5 — Nível 0: A Semente

> "Tudo começa com entropia verdadeira"

---

## Objetivo

Gerar uma seed BIP39 com entropia própria, sem confiar em nenhum software.
Adquirir os primeiros satoshis sem KYC.

**Tempo estimado:** 2-4 dias
**Dificuldade:** ★★☆☆☆

---

## Passo 0.1 — Comprar os materiais físicos

Antes de qualquer software, você precisa de coisas tangíveis.
Este é o fundamento sobre o qual tudo será construído.

### Lista de materiais

| Item | Opções | Custo estimado |
|------|--------|----------------|
| Dados de casino | 2 unidades, cores diferentes | R$ 10-20 |
| Placa de aço inox | Ou kit de arruelas (washers) | R$ 50-150 |
| Gravador elétrico | Ou estilete de metal | R$ 40-100 |
| Envelope pardo | Para papel intermediário | R$ 2 |

### Por que aço e não papel?

Papel queima. Papel molha. Papel rasga. Papel desbota.
Aço resiste a fogo (até ~1400°C), água, corrosão (se inox) e tempo.

Nunca use papel como backup permanente da sua seed.

---

## Passo 0.2 — Estudar o básico

Antes de gerar sua seed, dedique 1-2 horas para entender
estes conceitos fundamentais:

### O que são palavras BIP39?

BIP39 é um padrão que mapeia entropia (aleatoriedade)
para palavras humanas. A lista tem 2048 palavras padronizadas
em inglês. Cada palavra representa 11 bits de entropia.

24 palavras = 24 × 11 = 264 bits
(256 bits de entropia + 8 bits de checksum)

### Por que NUNCA gerar seed em software?

- **Computadores podem ser comprometidos:** malware pode roubar sua seed
- **RNG pode ser previsível:** geradores de números aleatórios de software
  podem ter falhas ou backdoors
- **Dados físicos = entropia verdadeira:** não há como prever
  o resultado de 100 lançamentos de dados

---

## Passo 0.3 — Gerar entropia com dados físicos

Este é o momento mais importante do Nível 0.
Faça com calma, em local privado, sem câmeras.

### Procedimento

1. Sente-se em local sem câmeras (celular longe)
2. Pegue os 2 dados de casino
3. Lance os dois simultaneamente
4. Anote EM PAPEL o resultado (ex: Dado vermelho=3, Dado azul=5 → anote "35")
5. Repita 100 vezes (50 pares de lançamentos)
6. Você terá 100 números entre 11 e 66

<div class="aviso">
⚠️ <strong>NUNCA:</strong>
- Filmar ou fotografar os lançamentos
- Ditar os números para um dispositivo
- Fazer perto de webcam ou celular
- Pular lançamentos ("99 está bom")
</div>

#!/bin/bash
# Script: gerar_livro.sh

CALIBRE="/usr/bin/calibre"
EBOOK_CONVERT="/usr/bin/ebook-convert"
FONTE="manuscrito/*.md"
SAIDA="saida"
TITULO="Guardião Soberano"
SUBTITULO="Da Semente ao Ecossistema Completo"

# Criar diretório de saída
mkdir -p "$SAIDA"

# Converter Markdown para EPUB
pandoc $FONTE \
    --from markdown \
    --to epub3 \
    --css recursos/stylesheet.css \
    --metadata title="$TITULO" \
    --metadata subtitle="$SUBTITULO" \
    --metadata author="Comunidade de Bitcoiners" \
    --metadata date="2026-06-23" \
    --metadata lang="pt-BR" \
    --toc \
    --toc-depth=3 \
    --epub-cover-image=recursos/cover.jpg \
    -o "$SAIDA/guardiao-soberano.epub"

# Converter EPUB → PDF
$EBOOK_CONVERT "$SAIDA/guardiao-soberano.epub" \
    "$SAIDA/guardiao-soberano.pdf" \
    --paper-size a5 \
    --pdf-page-margin-top 20 \
    --pdf-page-margin-bottom 20 \
    --pdf-page-margin-left 20 \
    --pdf-page-margin-right 20 \
    --pdf-default-font-size 12 \
    --pdf-mono-font-size 10 \
    --pdf-header-template "$TITULO" \
    --pdf-footer-template "_PAGENUM_"

# Converter EPUB → MOBI
$EBOOK_CONVERT "$SAIDA/guardiao-soberano.epub" \
    "$SAIDA/guardiao-soberano.mobi" \
    --mobi-file-type both

# Converter EPUB → AZW3
$EBOOK_CONVERT "$SAIDA/guardiao-soberano.epub" \
    "$SAIDA/guardiao-soberano.azw3"

echo "Livro gerado em: $SAIDA/"
ls -la "$SAIDA/"
