🪶 FEATHER WALLET NO TAILS: INSTALAÇÃO PERSISTENTE E SEGURA

"A Feather é o canivete suíço do Monero. No Tails, ela se torna o canivete invisível."

Você já sabe que a Feather é a carteira Monero leve e integrada ao Tor. Agora vamos ancorá‑la na persistência do Tails para que suas carteiras, histórico de transações e configurações sobrevivam a cada boot — sem jamais tocar o disco rígido do computador hospedeiro.

🎯 Objetivo

Instalar a Feather Wallet como AppImage na partição persistente, forçar o armazenamento dos dados da carteira nessa mesma partição, conectar via Tor (automático) e integrar tudo ao seu ritual de backup 3‑2‑1.

🧱 Pré‑requisitos

  • Tails iniciado com persistência desbloqueada e a opção "Personal Data" ativada (necessária para acessar /home/amnesia/Persistent/).
  • Pasta de aplicativos criada (se não existir, crie: mkdir -p /home/amnesia/Persistent/Apps).
  • Conexão Tor funcionando (o Tails já faz o trabalho pesado).

🛠️ Passo a passo

1. Baixar a AppImage oficial e verificar a assinatura

Obtenha a versão mais recente diretamente do site oficial, usando o navegador Tor ou o terminal com torsocks:

# Defina a versão (verifique em feather-wallet.org/download)
VERSION=2.7.0

# Baixe a AppImage e a assinatura .asc
torsocks wget -P /home/amnesia/Persistent/Apps/ \
  https://feather-wallet.org/files/releases/linux/feather-$VERSION-x86_64.AppImage \
  https://feather-wallet.org/files/releases/linux/feather-$VERSION-x86_64.AppImage.asc

Verifique a assinatura GPG (a chave pública do desenvolvedor está no site; no primeiro uso você precisará importá‑la). No terminal:

# Importar a chave pública (substituir pelo fingerprint real, ex: "tobtoht@featherwallet" etc.)
gpg --keyserver hkps://keys.openpgp.org --recv-keys 0x...  # ou use o arquivo .asc da chave
gpg --verify /home/amnesia/Persistent/Apps/feather-$VERSION-x86_64.AppImage.asc \
             /home/amnesia/Persistent/Apps/feather-$VERSION-x86_64.AppImage

Se a verificação for bem‑sucedida, o binário é autêntico. Torne‑o executável:

chmod +x /home/amnesia/Persistent/Apps/feather-$VERSION-x86_64.AppImage

2. Forçar a persistência dos dados da carteira

Por padrão, a Feather salva tudo em ~/.config/feather (no tmpfs, que evapora no desligamento). Vamos criar um script que aponte para uma pasta segura.

# Criar diretório para os dados da Feather na persistência
mkdir -p /home/amnesia/Persistent/feather-data

# Criar o script de inicialização
cat > /home/amnesia/Persistent/start-feather.sh << 'EOF'
#!/bin/bash
/home/amnesia/Persistent/Apps/feather-*-x86_64.AppImage \
    --datadir /home/amnesia/Persistent/feather-data \
    --use-tor &
EOF

chmod +x /home/amnesia/Persistent/start-feather.sh

Explicação:

  • --datadir força a Feather a usar o diretório persistente.
  • --use-tor garante que ela roteie tudo pelo Tor (embora o Tails já force a rede, essa flag reforça a configuração interna).

3. Primeira execução e criação da carteira

  1. Execute o script:

    /home/amnesia/Persistent/start-feather.sh
  2. A primeira tela perguntará sobre a conexão de rede. Escolha "Tor (recommended)". A Feather configurará automaticamente o proxy Tor do sistema (127.0.0.1:9050).
  3. Escolha o modo de carteira:
    • Create new wallet se for começar do zero.
    • Restore wallet from seed para importar uma seed existente.
  4. Ao criar uma nova carteira:
    • Dê um nome sugestivo (ex: fortaleza_quente).
    • Anote imediatamente as 25 palavras (seed) e o Restore Height que aparecerão na tela final. Escreva em papel resistente ou grave em metal. O Restore Height é o número do bloco no momento da criação (ex: 3185000). Ele acelera futuras restaurações.
  5. Continue. A Feather sincronizará com a blockchain via nó remoto sobre Tor. Em poucos minutos você terá seu saldo visível.

4. Verificar a localização dos dados

Após criar a carteira, feche a Feather e liste o conteúdo:

ls /home/amnesia/Persistent/feather-data/
# Você verá pastas como: wallets, logs, config.json

Isso confirma que as carteiras estão salvas na persistência LUKS.

Teste de sobrevivência:

  • Reinicie o Tails.
  • Desbloqueie a persistência.
  • Execute start-feather.sh.
  • Sua carteira deve reaparecer exatamente como estava, com histórico e saldo.

🧠 Dicas essenciais para o operador

  • Use a seed como último recurso, não como rotina. A Feather na persistência é cômoda, mas se o pendrive falhar ou for confiscado, a seed (guardada offline) é seu plano de fuga.
  • Restore Height é seu amigo. Anote‑o junto com a seed. Sem ele, uma restauração pode levar horas. Com ele, minutos.
  • Cuidado com o teclado em computadores públicos. Use o Onboard (teclado virtual) para digitar a seed na primeira restauração, se estiver em local suspeito.
  • Não reutilize a mesma senha de persistência para a carteira. Se você definir uma senha adicional para gastar fundos, escolha algo diferente.

🔗 Integração com o backup 3‑2‑1

Sua rotina de backup já deve estar incluindo a pasta feather-data. Atualize o script de backup:

tar czf - \
  /home/amnesia/Persistent/feather-data \
  /home/amnesia/Persistent/electrum-data \
  /home/amnesia/.local/share/haveno \
  /home/amnesia/Persistent/fortaleza_cofre.kdbx \
  /home/amnesia/Persistent/haveno_backup_amnesia.zip \
| gpg --symmetric --cipher-algo AES256 --output /media/backup/fortaleza_backup_$(date +%Y%m%d).tar.gz.gpg

A seed em si não deve estar dentro desse backup, a menos que você a armazene apenas no KeePassXC (que já está incluído, mas com senha separada). O ideal é que a seed física (papel/metal) fique em local geograficamente separado.

⚠️ Perigos e más práticas

  • Nunca execute a Feather sem o --datadir. Se você abrir a AppImage diretamente, ela criará ~/.config/feather no tmpfs. No próximo boot, sua carteira sumirá. Use sempre o script.
  • Não mantenha a Feather aberta durante a remoção do pendrive. Feche‑a antes de desligar para evitar corrupção do arquivo da carteira.
  • Atualizações da Feather: Ao baixar uma nova versão, substitua a AppImage antiga, mas não apague a pasta feather-data. Faça backup antes de atualizar.

Com a Feather ancorada na persistência, sua carteira Monero é tão portátil quanto seu pendrive e tão privada quanto a rede Tor. A cada boot, ela renasce pronta para negociar, sem deixar rastros no mundo físico.
