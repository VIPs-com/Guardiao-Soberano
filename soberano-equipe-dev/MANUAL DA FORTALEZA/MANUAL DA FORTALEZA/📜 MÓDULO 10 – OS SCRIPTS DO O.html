📜 MÓDULO 10 – OS SCRIPTS DO OPERADOR

"Cada linha que você escreve é um tijolo que você mesmo assentou. Nada de código estranho morando na sua fortaleza."

Você já tem as ferramentas. Agora você tem os scripts — pequenos feitiços de texto que transformam uma sequência de comandos complexos num clique. Eles eliminam a fadiga, previnem erros de digitação e garantem que você nunca inicie uma carteira no lugar errado.

Mas scripts são poder. E poder não verificado é um backdoor em potencial. Por isso, neste módulo você vai conhecer cada script usado na sua operação, entender linha por linha, e mantê-los em texto claro — legível, auditável, sob seu total controle.

🎯 Objetivo

Reunir, explicar e revisar todos os scripts criados ao longo do Manual da Fortaleza. Garantir que você saiba o que cada um faz e que nenhum código obscuro entre na sua persistência.

📦 Onde os scripts moram

Todos ficam dentro da partição persistente, na sua home:

/home/amnesia/Persistent/

Você pode criar uma subpasta scripts/ se preferir organização:

mkdir -p /home/amnesia/Persistent/scripts

Mas manter scripts de inicialização (start-*.sh) diretamente na raiz do Persistent facilita o acesso.

Regra de ouro: São arquivos de texto puro. Você os edita com o Mousepad (editor de texto do Tails) ou com nano. Jamais um binário compilado por terceiros.

📜 Inventário completo dos scripts

1. Script de inicialização da Feather Wallet

Arquivo: /home/amnesia/Persistent/start-feather.sh 

#!/bin/bash
/home/amnesia/Persistent/Apps/feather-*-x86_64.AppImage \
    --datadir /home/amnesia/Persistent/feather-data \
    --use-tor &
  • #!/bin/bash: indica que é um script Bash.
  • Caminho do AppImage com curinga * para não precisar atualizar a versão manualmente.
  • --datadir: força os dados da carteira na persistência, evitando a pasta volátil ~/.config/feather.
  • --use-tor: garante que a própria Feather use Tor (reforço, pois o Tails já força o tráfego).
  • & no final: executa em segundo plano e devolve o terminal.

✅ Revisão: Sempre que atualizar a Feather, confira se o novo AppImage ainda casa com o padrão feather-*-x86_64.AppImage; se mudarem o nome, ajuste o script.

2. Script de inicialização do Electrum

Arquivo: /home/amnesia/Persistent/start-electrum.sh 

#!/bin/bash
/home/amnesia/Persistent/Apps/electrum-*-x86_64.AppImage \
    --datadir /home/amnesia/Persistent/electrum-data "$@"
  • "$@": repassa argumentos extras que você quiser usar (ex: --testnet).
  • O resto é análogo ao da Feather: curinga no nome e --datadir para a persistência.

✅ Revisão: Verifique se a pasta electrum-data existe antes do primeiro uso. O Electrum a cria automaticamente, mas é bom saber.

3. Script de inicialização do KeePassXC

Arquivo: /home/amnesia/Persistent/start-keepass.sh 

#!/bin/bash
/home/amnesia/Persistent/Apps/KeePassXC-*.AppImage &

Simples e direto. O KeePassXC armazena o banco de dados onde você salvar, então apenas abrir o AppImage basta (você depois abrirá o arquivo .kdbxmanualmente ou por meio de "Abrir recente").

✅ Revisão: Certifique-se de que o AppImage do KeePassXC esteja com permissão de execução (chmod +x).

4. Script de inicialização do Haveno

Arquivo: /home/amnesia/Persistent/start-haveno.sh 

#!/bin/bash
/home/amnesia/Persistent/Apps/haveno/bin/Haveno \
    --data-dir=/home/amnesia/Persistent/haveno-data
  • Chama o binário Haveno de dentro da pasta extraída.
  • --data-dir aponta para a pasta persistente, garantindo que conta, contratos e chats sobrevivam.

✅ Revisão: Se o Haveno mudar o nome do executável ou o caminho, atualize. Mantenha o backup do arquivo .zip da conta Haveno em local separado.

5. Script de backup 3-2-1 (Toolkit)

Arquivo: /home/amnesia/Persistent/fortaleza-toolkit.sh

Esse já foi detalhado no Módulo 9. Ele oferece um menu para:

  • Verificar assinatura PGP de um arquivo.
  • Criar backup criptografado.
  • Restaurar backup.

O código é maior, mas integralmente em Bash. Você deve revisar linha por linha ao copiar para seu Tails.

✅ Revisão: Confirme que os caminhos das pastas incluídas no backup (feather-data, electrum-data, haveno, etc.) realmente existem e são os que você usa. Adapte se tiver pastas extras.

6. Script de atualização de aplicativos com verificação

Arquivo: /home/amnesia/Persistent/atualizar-fortaleza.sh 

#!/bin/bash
PASTA_APPS="/home/amnesia/Persistent/Apps"
cd $PASTA_APPS

# Feather
echo "Baixando Feather..."
torsocks wget -q https://featherwallet.org/files/releases/linux/feather-2.7.0.AppImage
torsocks wget -q https://featherwallet.org/files/releases/linux/feather-2.7.0.AppImage.asc
gpg --verify feather-2.7.0.AppImage.asc feather-2.7.0.AppImage

# Electrum
echo "Baixando Electrum..."
torsocks wget -q https://download.electrum.org/4.5.5/electrum-4.5.5-x86_64.AppImage
torsocks wget -q https://download.electrum.org/4.5.5/electrum-4.5.5-x86_64.AppImage.asc
gpg --verify electrum-4.5.5-x86_64.AppImage.asc electrum-4.5.5-x86_64.AppImage

# KeePassXC
echo "Baixando KeePassXC..."
torsocks wget -q https://github.com/keepassxreboot/keepassxc/releases/latest/download/KeePassXC-2.7.9-x86_64.AppImage
torsocks wget -q https://github.com/keepassxreboot/keepassxc/releases/latest/download/KeePassXC-2.7.9-x86_64.AppImage.asc
gpg --verify KeePassXC-2.7.9-x86_64.AppImage.asc KeePassXC-2.7.9-x86_64.AppImage

echo "Se nenhuma linha mostrar 'Assinatura incorreta', tudo OK."

✅ Revisão: As URLs e versões devem ser atualizadas manualmente por você quando novas versões saírem. Não confie em um script que baixa sempre o "latest" sem verificar a assinatura (a URL do KeePassXC usa /latest/, mas a versão pode mudar e a assinatura também; melhor fixar versões e conferir no site).

7. Script de destruição segura da persistência (opcional, para emergências)

Arquivo: /home/amnesia/Persistent/limpar-persistencia.sh 

#!/bin/bash
echo "ATENÇÃO: Isso apagará permanentemente a partição persistente!"
echo "Digite 'APAGAR TUDO' para confirmar:"
read CONFIRM
if [ "$CONFIRM" = "APAGAR TUDO" ]; then
    sudo shred -v -n 1 -z /dev/sdX2
    echo "Persistência destruída."
else
    echo "Cancelado."
fi

Substitua /dev/sdX2 pelo dispositivo correto da sua partição persistente (TailsData). Use lsblk para conferir.

✅ Revisão: Este script é perigoso. Só o mantenha se souber o que está fazendo e jamais execute sem certeza.

🧠 Por que texto claro e não binários?

  • Transparência total: Você pode abrir cada script com um editor e ver exatamente o que ele faz. Nenhuma surpresa.
  • Auditável: Se você esquecer um detalhe, a leitura do script refresca a memória.
  • Editável: Pode ajustar caminhos, adicionar pastas ao backup, mudar opções sem depender de ninguém.
  • Resistente: Um .sh não fica "corrompido" como um binário; se o pendrive tiver um erro, você consegue recriar o script facilmente a partir do seu caderno de anotações (ou deste Manual).

Nunca baixe scripts prontos de fontes duvidosas. Digite-os você mesmo ou copie do Manual e leia cada linha enquanto copia. Isso é um mantra cypherpunk.

🛡️ Revisão e manutenção periódica

A cada mês, ou antes de uma viagem, faça:

⚠️ Perigos

  • Esquecer de tornar executável: Sem chmod +x, o script não roda. O Tails não executa scripts clicados diretamente; use o terminal ou crie atalhos .desktop (avançado).
  • Curinga * enganoso: Se houver mais de um AppImage na pasta (ex: versão antiga e nova), o curinga pode pegar a errada. Mantenha apenas a versão atual.
  • Script de backup com read -s: A senha não aparece, mas fica armazenada na memória do bash. Após usar, feche o terminal ou digite history -c.
  • Não use sudo desnecessariamente: Scripts que precisam de sudo devem ser revisados com atenção redobrada, pois podem danificar o sistema ou a persistência.

🧪 Exercício: Revise cada script agora mesmo

Abra o Mousepad no Tails e carregue cada um dos seus scripts. Leia em voz alta (ou mentalmente) cada linha, imaginando o que o computador fará. Se houver algo que você não entende, volte ao módulo correspondente. O conhecimento do próprio código é a última camada da soberania.
