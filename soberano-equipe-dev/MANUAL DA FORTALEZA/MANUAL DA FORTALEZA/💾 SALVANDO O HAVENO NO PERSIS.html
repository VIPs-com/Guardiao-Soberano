💾 SALVANDO O HAVENO NO PERSISTENT STORAGE DO TAILS

"No Tails, tudo que não está na persistência vira fumaça no reboot."

Por padrão, o Haveno guarda seus dados em ~/.local/share/haveno — que fica no sistema de arquivos temporário (tmpfs) do Tails. Ao desligar, tudo some: conta, contratos, chats, reputação, ordens abertas. Para transformar o Haveno em algo usável entre sessões, você precisa movê-lo para dentro da partição persistente criptografada.

🎯 Objetivo

Fazer com que o Haveno leia e escreva seus dados em /home/amnesia/Persistent/haveno-data (ou similar), que sobrevive a reinicializações e fica protegido pela senha LUKS.

🧱 Pré-requisitos

  • Tails iniciado com persistência desbloqueada e a opção "Dotfiles" ou "Personal Data" ativada (na verdade, usaremos a pasta Persistent diretamente, que já está disponível com qualquer persistência).
  • Haveno já baixado e extraído (ex: em /home/amnesia/Persistent/Apps/haveno).
  • Terminal aberto.

🛠️ Método 1: Link Simbólico (recomendado para iniciantes)

Esse método engana o Haveno, fazendo-o acreditar que está escrevendo no local padrão, mas os dados vão para a persistência.

Passo 1: Primeira execução para gerar a pasta original

Execute o Haveno uma vez para que ele crie ~/.local/share/haveno. Depois feche completamente o Haveno.

Passo 2: Mover a pasta para a persistência

mv ~/.local/share/haveno /home/amnesia/Persistent/haveno-data

Agora a pasta original não existe mais; ela foi rebatizada e movida para o local seguro.

Passo 3: Criar o link simbólico

ln -s /home/amnesia/Persistent/haveno-data ~/.local/share/haveno

Verifique:

ls -l ~/.local/share/haveno
# Deve mostrar: haveno -> /home/amnesia/Persistent/haveno-data

Passo 4: Testar

Abra o Haveno novamente. Sua conta deve estar lá. Faça um teste rápido (abrir uma oferta, enviar uma mensagem de chat) e reinicie o Tails. Depois, abra novamente o Haveno. Tudo deve ter sobrevivido.

🛠️ Método 2: Argumento --data-dir (mais limpo, mas você precisa lembrar de usar sempre)

Em vez de link, você pode iniciar o Haveno apontando diretamente para a pasta persistente.

Passo 1: Criar a pasta de dados na persistência

mkdir -p /home/amnesia/Persistent/haveno-data

Passo 2: Iniciar o Haveno com o caminho personalizado

/home/amnesia/Persistent/Apps/haveno/bin/Haveno --data-dir=/home/amnesia/Persistent/haveno-data

Na primeira execução, o Haveno criará todos os arquivos lá dentro.

Passo 3: Criar um script de inicialização para não esquecer

Crie um arquivo ~/Persistent/start-haveno.sh:

#!/bin/bash
/home/amnesia/Persistent/Apps/haveno/bin/Haveno --data-dir=/home/amnesia/Persistent/haveno-data

Torne-o executável:

chmod +x /home/amnesia/Persistent/start-haveno.sh

A partir de agora, sempre inicie o Haveno pelo script.

🧠 Dicas Cruciais para o Operador

1. Cuidado com o backup da conta Haveno

Além da pasta de dados, o Haveno oferece um backup de conta específico (Account → Backup). Esse arquivo .zip contém as chaves da sua conta de negociação e deve ser armazenado com segurança extra.
Salve-o diretamente na persistência:

cp ~/Downloads/haveno_backup_amnesia.zip /home/amnesia/Persistent/

E inclua esse arquivo no seu backup 3-2-1 (Módulo 4).

2. Atualizações do Haveno

Se você baixar uma nova versão do Haveno, basta substituir os arquivos em /home/amnesia/Persistent/Apps/haveno. Os dados na pasta persistente (haveno-data) são compatíveis entre versões próximas. Mesmo assim, faça backup antes de atualizar.

3. Permissões e dono

No Tails, você sempre opera como usuário amnesia. Os arquivos na persistência pertencem a amnesia automaticamente. Se por algum motivo houver erro de permissão:

sudo chown -R amnesia:amnesia /home/amnesia/Persistent/haveno-data

4. Economia de espaço

A pasta haveno-data pode crescer com logs e dados de blockchain local (se você estiver rodando um nó Monero junto, o que não é recomendado no Tails). Use a opção de usar um nó remoto na Feather e na Haveno para evitar baixar a blockchain inteira.

🧪 Teste de Sobrevivência

  1. Configure o Haveno em um boot do Tails.
  2. Faça uma pequena operação (ex: crie uma oferta de teste).
  3. Reinicie o computador.
  4. Inicie o Tails, desbloqueie a persistência.
  5. Execute o Haveno (via script ou normalmente, se usou o link).
  6. Verifique se a oferta ainda está lá.

Se sim, sua DEX pessoal está oficialmente ancorada na persistência criptografada.

⚠️ Perigos

  • Nunca formate ou recrie a persistência sem ter um backup externo da pasta haveno-data e do arquivo de backup da conta.
  • Se você esquecer de desbloquear a persistência e iniciar o Haveno, ele pode criar uma pasta vazia em ~/.local/share/haveno (tmpfs). Nesse caso, feche o Haveno, apague essa pasta vazia e recrie o link simbólico, ou simplesmente reinicie o Tails com a persistência ativada corretamente.
  • Não armazene a seed da sua carteira Monero dentro do Haveno ou da pasta de dados. O local da seed é papel/aço, ou no KeePassXC — separado.

Com essa configuração, sua identidade de negociação e seu histórico sobrevivem a reinicializações, quedas de energia e à natureza amnésica do Tails. A fortaleza agora tem memória.
