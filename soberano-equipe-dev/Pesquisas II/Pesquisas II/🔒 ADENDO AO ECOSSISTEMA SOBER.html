1. ABCTracer 2025

O que é

ABCTracer é o nome dado pela comunidade a uma classe de ataques de correlação temporal que evoluíram em 2025. Não é uma ferramenta única, mas uma metodologia de análise que combina:

  • Timing analysis: Correlacionar horários de transações entre blockchains
  • Amount correlation: Valores similares entrando e saindo em janelas curtas
  • Cross-chain fingerprinting: Metadados que vazam entre BTC e XMR

Como funciona o ataque

┌─────────────────────────────────────────────────────────────┐
│              ABCTRACER — CENÁRIO DE ATAQUE                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ⚠️ O QUE O ATACANTE VÊ:                                   │
│                                                             │
│  Blockchain Bitcoin (pública):                              │
│  • UTXO de 0.05 BTC entra em swap às 14:32                  │
│  • Transação de swap confirmada às 14:45                    │
│                                                             │
│  Blockchain Monero (privada, mas...):                       │
│  • Output de ~0.98 XMR aparece às 15:10                     │
│  • Valor convertido ≈ 0.0485 BTC (muito próximo!)           │
│                                                             │
│  ⚠️ CORRELAÇÃO:                                            │
│  • Intervalo: 38 minutos                                    │
│  • Valor: ~97% do original (menos fee)                      │
│  • Probabilidade de ser o mesmo swap: ALTA                  │
│                                                             │
│  🔓 PRIVACIDADE DO MONERO QUEBRADA                          │
│  (para este swap específico)                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Contra-medidas (já integradas ao seu protocolo)

Pesquisa adicional

  • OXT Research (oxt.me) — Ferramentas de análise de blockchain
  • Monero Research Lab — Papers sobre correlação cross-chain
  • KYCP.org — Simulador de ataques de correlação (não use com dados reais!)

2. JoinMarket

O que é

JoinMarket é uma implementação de CoinJoin que usa um mercado descentralizado de liquidez. Diferente do Whirlpool (coordinator central), no JoinMarket:

  • Makers oferecem BTC para misturar (ganham fees)
  • Takers iniciam os coinjoins (pagam fees)
  • Não há coordinator central — é P2P via rede Tor

Comparação JoinMarket vs Whirlpool

Instalação Básica no Whonix

# Na Whonix Workstation
git clone https://github.com/JoinMarket-Org/joinmarket-clientserver.git
cd joinmarket-clientserver

# Instalar dependências
pip3 install -r requirements.txt

# Gerar carteira
python3 wallet-tool.py generate

# Configurar para Tor
nano joinmarket.cfg
# [MESSAGING:server]
# host = 127.0.0.1
# port = 9050
# type = socks5

Quando considerar JoinMarket

  • Se o coordinator Whirlpool ficar offline por dias
  • Se você quiser valores de coinjoin personalizados (não apenas pools fixos)
  • Se quiser GANHAR fees como maker (fornecendo liquidez)
  • Como alternativa de backup ao Whirlpool

3. PayJoin (Pay-to-EndPoint)

O que é

PayJoin (também chamado P2EP ou Pay-to-EndPoint) é uma transação colaborativa onde o pagador e o recebedor contribuem inputs para a mesma transação, quebrando a heurística de "common input ownership".

Como funciona

┌─────────────────────────────────────────────────────────────┐
│                 PAYJOIN — ANTES E DEPOIS                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  TRANSAÇÃO NORMAL:                                          │
│  Input: 0.05 BTC (seu) → Output: 0.03 BTC (pagamento)      │
│                          → Output: 0.0199 BTC (troco)       │
│                                                             │
│  🔍 Analista vê: 1 pessoa gastou 0.05, recebeu 0.0199       │
│                                                             │
│  ─────────────────────────────────────────────────          │
│                                                             │
│  PAYJOIN:                                                   │
│  Input: 0.05 BTC (seu) + 0.02 BTC (do recebedor)           │
│  → Output: 0.03 BTC (pagamento efetivo)                     │
│  → Output: 0.0398 BTC (volta pra você)                      │
│                                                             │
│  🔍 Analista vê: 2 pessoas consolidaram inputs              │
│  → Não sabe quem pagou quem                                 │
│  → Não sabe qual output é troco                             │
│  → Heurística de propriedade comum QUEBRADA                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Ferramentas que suportam PayJoin

PayJoin + Whirlpool (Estratégia Combinada)

  1. Antes do Whirlpool: Use PayJoin para compras normais (quebra heurística inicial)
  2. Depois do PayJoin: Mova o troco para Whirlpool (anonset começa mais alto)
  3. Após Whirlpool: Use PayJoin para gastar outputs limpos (adiciona camada extra)

4. BIP47 — Códigos de Pagamento Reutilizáveis

O que é

BIP47 (Payment Codes) permite publicar um "código de pagamento" reutilizável que gera endereços únicos para cada transação, sem interação entre as partes.

Por que é relevante

No seu ecossistema, você pode publicar um código BIP47 para receber pagamentos privados sem:

  • Reutilizar endereços (má prática)
  • Gerar novos endereços manualmente a cada vez
  • Usar xpub publicamente (expõe todo histórico)

Ferramentas BIP47

Estado atual (2026)

O BIP47 perdeu força após a apreensão do Samourai Wallet (2024), mas o padrão é aberto e implementável. O Sparrow está adicionando suporte incremental.

5. Ricochet — Envio com Saltos

O que é

Ricochet (ou "hopping") é uma técnica de enviar BTC através de múltiplos endereços próprios antes do destino final, criando distância on-chain.

Exemplo

Sua carteira → Endereço A → Endereço B → Endereço C → Destino final
                                                         
Cada salto = 1 transação = custo de fee

Quando usar

  • NÃO use com outputs pós-coinjoin (diminui anonimato ao consolidar)
  • Use com outputs não-misturados quando não puder usar Whirlpool
  • Eficácia é baixa contra análise moderna (analistas já modelam ricochet)

Alternativa superior

Em vez de Ricochet, prefira:

  1. CoinJoin (Whirlpool/JoinMarket) — anonimato real
  2. PayJoin — quebra heurística sem custo extra de fees
  3. Swap XMR — anonimato de rede diferente

6. Coin Control Avançado

O que é

Coin Control é a prática de selecionar manualmente quais UTXOs gastar em cada transação. O Sparrow oferece isso na aba UTXOs.

Regras de Ouro

┌─────────────────────────────────────────────────────────────┐
│              COIN CONTROL — REGRAS                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ✅ FAÇA:                                                   │
│  • Gastar 1 UTXO por vez (quando possível)                  │
│  • Rotular UTXOs por origem (KYC, coinjoin, swap)          │
│  • Usar UTXOs pós-coinjoin SEPARADOS de não-misturados     │
│  • Congelar UTXOs "sujos" para não gastar sem querer        │
│                                                             │
│  ❌ NÃO FAÇA:                                               │
│  • Consolidar 2+ UTXOs pós-coinjoin (reduz anonimato)      │
│  • Misturar UTXO KYC com UTXO coinjoin (contaminação)      │
│  • Gastar dust (valores < fee) — polui carteira            │
│  • Usar "Send Max" sem verificar quais UTXOs selecionados  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Como Congelar UTXOs no Sparrow

  1. Aba UTXOs
  2. Clique com botão direito no UTXO
  3. Freeze → UTXO fica cinza, não será gasto
  4. Para descongelar: Unfreeze

8. Samourai Whirlpool — História e Sucessor

O que aconteceu

Em Abril de 2024, o Departamento de Justiça dos EUA apreendeu os servidores do Samourai Wallet e prendeu os fundadores (Keonne Rodriguez e William Lonergan Hill). O coordinator Whirlpool original foi desligado.

O que mudou

Sucessor atual

O Sparrow Wallet é o sucessor de facto para Whirlpool. Ele:

  • Conecta ao coordinator comunitário
  • Mantém o protocolo Whirlpool original (ZeroLink)
  • É mantido por Craig Raw (desenvolvedor independente, baseado no Canadá)
  • Tem auditoria de segurança contínua

O que NÃO usar

9. Ferramentas de Análise de Privacidade

Para auditar sua própria privacidade

Como auto-auditar

  1. Pegue um UTXO pós-coinjoin
  2. Cole o TXID no OXT Explorer (via Tor!)
  3. Veja o "cluster" — quantos endereços estão ligados?
  4. Se o cluster for pequeno (só seus endereços), o anonimato é baixo
  5. Se o cluster for grande (muitos participantes do round), anonimato é alto
