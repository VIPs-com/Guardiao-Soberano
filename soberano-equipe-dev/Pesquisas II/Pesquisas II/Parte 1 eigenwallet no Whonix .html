Parte 1: eigenwallet no Whonix — Swaps BTC↔XMR direto da Workstation

Perfeito. Vamos fechar o ciclo completo com duas configurações de alto nível: eigenwallet no Whonix (para swaps BTC↔XMR direto da Workstation) e EPS com Tor hidden service (para eliminar dependência de servidores Electrum públicos). No final, você terá um ambiente Whonix totalmente autônomo.

Parte 1: eigenwallet no Whonix — Swaps BTC↔XMR direto da Workstation

Pré-requisitos específicos

  • Whonix Workstation já configurada (como no guia anterior)
  • Sparrow Wallet funcionando com carteira watching-only
  • Feather Wallet instalada na Workstation
  • Pelo menos um output BTC pós-coinjoin disponível
  • Coldcard pronto para assinar PSBTs
  • 2-3 horas para o primeiro swap (entre configuração e execução)

Passo 1 — Entender o modelo de ameaça do eigenwallet

O eigenwallet é um cliente de atomic swap on-chain. Isso significa que:

  • A transação Bitcoin do swap é pública na blockchain — visível para qualquer um
  • A transação Monero é privada, mas o vínculo entre o UTXO BTC de entrada e o XMR de saída pode ser inferido se você não tomar cuidados
  • Por isso você só deve alimentar o eigenwallet com BTC pós-coinjoin — UTXOs com anonset ≥ 5
  • O eigenwallet atua como "taker" (você pega ofertas de "makers" que já anunciaram)

No Whonix, o tráfego de rede do eigenwallet sai pelo Tor automaticamente (herdado do Gateway). Isso é bom, mas não suficiente — você precisa garantir que o swap não vaze metadados.

Passo 2 — Instalar e verificar o eigenwallet

2.1 Download e verificação PGP

Na Whonix Workstation:

# Criar diretório para ferramentas
mkdir -p ~/Applications
cd ~/Applications

# Baixar eigenwallet
wget https://eigenwallet.org/releases/latest/eigenwallet-linux.AppImage

# Baixar assinatura
wget https://eigenwallet.org/releases/latest/eigenwallet-linux.AppImage.asc

# Importar chave do binarybaron (verificar fingerprint em múltiplas fontes!)
curl https://eigenwallet.org/binarybaron.asc | gpg --import

# Verificar fingerprint (deve bater com a publicada no Twitter/GitHub/Matrix do dev)
gpg --fingerprint binarybaron

# Verificar assinatura
gpg --verify eigenwallet-linux.AppImage.asc eigenwallet-linux.AppImage

# Tornar executável
chmod +x eigenwallet-linux.AppImage

2.2 Criar lançador para conveniência

Crie um script ~/start_eigenwallet.sh:

#!/bin/bash
echo "Verificando Tor..."
curl --socks5-hostname 127.0.0.1:9050 https://check.torproject.org 2>/dev/null | grep -q "Congratulations" && echo "Tor OK" || echo "TOR FALHOU - ABORTE"
echo "Iniciando eigenwallet..."
~/Applications/eigenwallet-linux.AppImage &
chmod +x ~/start_eigenwallet.sh

Passo 3 — Configurar o ambiente de swap

3.1 Estrutura de carteiras no Sparrow

Você vai precisar de três "zonas" na sua carteira Sparrow:

Por que separar: Se o swap falhar ou você precisar cancelar, o UTXO volta para Swap_Ready e você não contamina os outros UTXOs pós-coinjoin.

Todas as três carteiras usam a mesma xpub do Coldcard — são só visões diferentes dos mesmos fundos.

3.2 Preparar um UTXO para swap

  1. Na carteira Postmix_Whonix, selecione um UTXO com anonset ≥ 5 e valor adequado (ex: 0.01 BTC para teste)
  2. Botão direito → Send to → Swap_Ready_Whonix (carteira de destino)
  3. Crie PSBT, assine no Coldcard, transmita
  4. Aguarde 1 confirmação

Agora o UTXO está isolado e rotulado para swap.

Passo 4 — Gerar endereços para o swap

4.1 Endereço XMR (recebimento)

No Feather Wallet (aberto na mesma Workstation):

  • Receive → New address
  • Anote o subendereço no KeePassXC com label: Swap eigen 2026-06-23 #1

4.2 Endereço BTC de change (troco)

O eigenwallet precisa de um endereço Bitcoin para onde enviar o troco (se o valor do swap for menor que o UTXO de entrada).

No Sparrow, carteira Swap_Ready_Whonix:

  • Aba Addresses → selecione um endereço nunca usado com derivação BIP84
  • Copie o endereço
  • Anote no KeePassXC: Change eigen swap 2026-06-23 #1

Crítico: Use SEMPRE um endereço virgem. Reutilizar endereço de change queima privacidade e pode invalidar o anonimato do coinjoin.

Passo 5 — Executar o swap no eigenwallet

5.1 Iniciar o eigenwallet

~/start_eigenwallet.sh

O eigenwallet abrirá com interface gráfica (Electron). Ele automaticamente usa Tor (herda do sistema Whonix).

5.2 Escolher o par e direção

  • Selecione: BTC → XMR (você está vendendo BTC, comprando XMR)
  • O eigenwallet mostrará uma lista de "makers" (ofertas disponíveis)

5.3 Analisar as ofertas

Cada maker mostra:

Escolha um maker com:

  • Pelo menos 10 swaps completos
  • Bond alto (mostra compromisso)
  • Preço dentro do mercado
  • Valor compatível com seu UTXO

5.4 Inserir os dados do swap

O eigenwallet pedirá:

  1. Bitcoin Amount: Valor do UTXO que você vai usar (ex: 0.01 BTC)
  2. Bitcoin Address (Refund): O endereço de change que você gerou no Sparrow
  3. Monero Address: O subendereço do Feather Wallet

Revise três vezes. Um erro no endereço Monero = perda permanente dos fundos.

5.5 Criar a transação Bitcoin do swap

O eigenwallet gerará uma transação Bitcoin que bloqueia seus fundos num contrato atômico.

Você verá: "Transaction created. Please sign with your wallet."

5.6 Assinar com Coldcard (PSBT via pasta compartilhada)

O eigenwallet exporta a transação como PSBT ou transação raw. Você precisa:

  1. Salvar o PSBT na pasta compartilhada: /mnt/psbt_bridge/swap_psbt.psbt
  2. No host, copiar para o MicroSD
  3. Coldcard → Ready to Sign → selecionar swap_psbt.psbt → assinar
  4. Copiar PSBT assinado de volta para /mnt/psbt_bridge/swap_signed.psbt
  5. No eigenwallet, carregar a transação assinada

Alternativa: Se você configurou USB passthrough, pode conectar o Coldcard direto na VM e assinar sem MicroSD. Mas o método MicroSD é mais seguro (air-gapped).

5.7 Transmitir e aguardar

Após carregar a transação assinada, o eigenwallet:

  1. Transmite a transação Bitcoin (via Tor)
  2. Monitora a blockchain Monero (via Tor)
  3. Aguarda o maker cumprir a parte dele

Tempo típico: 25-90 minutos, dependendo:

  • Congestionamento da rede Bitcoin
  • Confirmações necessárias (geralmente 10 confirmações Bitcoin + 10 confirmações Monero)
  • Velocidade do maker

5.8 Confirmar recebimento

No Feather Wallet:

  • Aguardar aparecer o XMR (com 10 confirmações Monero, ~20 min)
  • Verificar se o valor bate (menos a taxa do maker, geralmente 1-3%)

Passo 6 — Pós-swap: intervalo de segurança e limpeza

6.1 Intervalo mínimo obrigatório

Como você bem sabe: nunca mova o XMR imediatamente após o swap.

  • Mínimo: 24 horas
  • Ideal: 3-7 dias, período em que você pode fazer OUTROS swaps com outros UTXOs
  • Durante esse intervalo, o XMR fica parado no subendereço do Feather

6.2 Churn Monero (opcional mas recomendado)

Após o intervalo, para quebrar qualquer possível correlação temporal residual:

  1. No Feather, crie uma nova carteira (seed nova, derivada na hora)
  2. Envie o XMR da carteira do swap para essa nova carteira
  3. Aguarde mais 1 hora
  4. Agora o XMR está "limpo" para uso

6.3 Registrar o swap no KeePassXC

Crie uma entrada detalhada:

=== SWAP EIGENWALLET ===
Data: 2026-06-23 15:30 UTC
Maker: (onion do maker ou ID)
UTXO BTC entrada: txid:index (0.01 BTC, anonset 5)
Endereço BTC change: bc1q...
Subendereço XMR: 8...
Txid swap BTC: abc...
Txid recebimento XMR: def...
Valor recebido XMR: 0.42 XMR
Fee maker: 2%
Status: Concluído

Parte 2: EPS com Tor Hidden Service — Ciclo Completo

Agora vamos configurar o Electrum Personal Server (EPS) com um Tor hidden service, para que seu Sparrow (no Whonix ou no Tails) se conecte diretamente ao SEU node Bitcoin, sem intermediários.

Arquitetura do sistema completo

┌─────────────────────────────────────────────────────────┐
│                     HOST FÍSICO                         │
│                                                         │
│  ┌──────────────────┐  ┌──────────────────────────┐    │
│  │ Whonix-Gateway   │  │ VM Node Bitcoin + EPS    │    │
│  │ (Tor, rede)      │  │                          │    │
│  │                  │  │ - Bitcoin Core (pruned)  │    │
│  │ Rede interna:    │  │ - EPS                    │    │
│  │ 10.152.152.0/24  │  │ - Tor hidden service     │    │
│  └────────┬─────────┘  │ - .onion: xyz.onion      │    │
│           │            └────────────┬─────────────┘    │
│           │                         │                   │
│  ┌────────┴─────────┐              │                   │
│  │ Whonix-Workstation│              │                   │
│  │                   │              │                   │
│  │ - Sparrow         │──────────────┘                   │
│  │ - eigenwallet     │  Conexão via .onion              │
│  │ - Feather         │  (passa pelo Gateway)            │
│  │ - KeePassXC       │                                  │
│  └───────────────────┘                                  │
│                                                         │
│  ┌──────────────────┐                                   │
│  │ Pasta compart.   │                                   │
│  │ /psbt_bridge     │ ← Coldcard troca PSBTs aqui       │
│  └──────────────────┘                                   │
└─────────────────────────────────────────────────────────┘

Pré-requisitos para o EPS

  • Bitcoin Core instalado e sincronizado (pode ser no host, numa VM separada, ou num Raspberry Pi)
  • 200+ GB livres para blockchain pruned (ou 600+ GB para full node)
  • Python 3.8+
  • Git

Opção de hardware para o node

Vou assumir VM separada no mesmo host (Debian minimal, só Bitcoin Core + EPS + Tor).

Passo 1 — Criar VM para o Node Bitcoin

1.1 Criar VM Debian minimal

No VirtualBox:

  • New VM → nome: Bitcoin-Node
  • RAM: 4096 MB (8192 MB se full node)
  • Disco: 400 GB dinâmico (pruned 10 GB) ou 800 GB (full)
  • Rede: NAT (para baixar blockchain inicial) + Internal Network Whonix (para comunicação com Workstation)

1.2 Instalar Debian e dependências

# No Debian minimal
sudo apt update && sudo apt upgrade -y
sudo apt install git python3 python3-pip tor ufw -y

# Criar usuário dedicado
sudo useradd -m -s /bin/bash bitcoin
sudo usermod -aG sudo bitcoin

Passo 2 — Instalar Bitcoin Core

# Como usuário bitcoin
su - bitcoin

# Baixar e verificar Bitcoin Core
wget https://bitcoincore.org/bin/bitcoin-core-27.0/bitcoin-27.0-x86_64-linux-gnu.tar.gz
wget https://bitcoincore.org/bin/bitcoin-core-27.0/SHA256SUMS
wget https://bitcoincore.org/bin/bitcoin-core-27.0/SHA256SUMS.asc

# Verificar (importar chaves dos devs Bitcoin Core)
gpg --keyserver hkps://keys.openpgp.org --recv-keys ...
gpg --verify SHA256SUMS.asc SHA256SUMS
sha256sum --check SHA256SUMS --ignore-missing

# Instalar
tar -xzf bitcoin-27.0-x86_64-linux-gnu.tar.gz
sudo install -m 0755 -o root -g root -t /usr/local/bin bitcoin-27.0/bin/*

# Criar diretório de dados
mkdir -p ~/.bitcoin

2.1 Configurar bitcoin.conf

nano ~/.bitcoin/bitcoin.conf
# Configuração para node pruned + Tor
prune=10000
txindex=0

# Conexão via rede interna Whonix (Gateway)
proxy=10.152.152.10:9050

# Escutar na rede interna para EPS
rpcbind=127.0.0.1
rpcallowip=127.0.0.1

# Autenticação RPC
rpcauth=seuuser:hash_gerado_com_rpcauth.py

# Não sair para clearnet
onlynet=onion
listen=1
listenonion=1

# Forçar Tor
dnsseed=0
dns=0

Gerar rpcauth: 

wget https://raw.githubusercontent.com/bitcoin/bitcoin/master/share/rpcauth/rpcauth.py
python3 rpcauth.py seuuser
# Copiar a linha rpcauth= gerada para o bitcoin.conf

2.2 Iniciar Bitcoin Core e sincronizar

bitcoind -daemon
tail -f ~/.bitcoin/debug.log

A sincronização inicial pode levar 2-7 dias via Tor. Para acelerar, você pode sincronizar via clearnet primeiro e depois ativar onlynet=onion.

Passo 3 — Instalar EPS (Electrum Personal Server)

3.1 Clonar e configurar

su - bitcoin
cd ~
git clone https://github.com/chris-belcher/electrum-personal-server.git
cd electrum-personal-server

# Instalar dependências
pip3 install -r requirements.txt

# Criar arquivo de configuração
cp config.ini.sample config.ini
nano config.ini

3.2 Configuração do EPS

[master-public-keys]
# Adicione sua xpub aqui (a mesma do Coldcard que você usa no Sparrow)
sua_carteira = xpub6...

[bitcoin-rpc]
host = 127.0.0.1
port = 8332
user = seuuser
password = senha_gerada_pelo_rpcauth

# Intervalo de polling (mais baixo = mais rápido, mais CPU)
poll_interval_listening = 30
poll_interval_connected = 5

[electrum-server]
host = 127.0.0.1
port = 50002
use_ssl = no

# Tor
tor_host = 127.0.0.1
tor_port = 9050
tor_hidden_service = yes

Importante: O EPS só escuta transações relacionadas à sua xpub. Ele não expõe dados de outros endereços.

3.3 Primeira execução e teste

python3 electrum-personal-server.py config.ini

O EPS fará um scan inicial da blockchain (pode levar 1-2 horas) para indexar as transações da sua carteira.

Logs de sucesso:

INFO:electrumpersonalserver:Starting Electrum Personal Server
INFO:electrumpersonalserver:Listening on 127.0.0.1:50002
INFO:electrumpersonalserver:Tor hidden service: xyz.onion:50002

Anote o endereço .onion gerado. Ele estará em:

cat /var/lib/tor/electrum_hidden_service/hostname

Passo 4 — Configurar o Tor hidden service (se não automático)

Se o EPS não gerou o .onion automaticamente, configure manualmente:

sudo nano /etc/tor/torrc

Adicione:

HiddenServiceDir /var/lib/tor/electrum_hidden_service/
HiddenServicePort 50002 127.0.0.1:50002
sudo systemctl restart tor
sudo cat /var/lib/tor/electrum_hidden_service/hostname
# Exemplo: abcdef123456.onion

Passo 5 — Criar serviços systemd para Bitcoin Core e EPS

Para que tudo inicie automaticamente no boot da VM:

5.1 Serviço Bitcoin Core

sudo nano /etc/systemd/system/bitcoind.service
[Unit]
Description=Bitcoin Core
After=network.target

[Service]
User=bitcoin
Group=bitcoin
Type=forking
ExecStart=/usr/local/bin/bitcoind -daemon
ExecStop=/usr/local/bin/bitcoin-cli stop
Restart=on-failure
RestartSec=30

[Install]
WantedBy=multi-user.target

5.2 Serviço EPS

sudo nano /etc/systemd/system/eps.service
[Unit]
Description=Electrum Personal Server
After=bitcoind.service tor.service
Wants=bitcoind.service tor.service

[Service]
User=bitcoin
Group=bitcoin
WorkingDirectory=/home/bitcoin/electrum-personal-server
ExecStart=/usr/bin/python3 /home/bitcoin/electrum-personal-server/electrum-personal-server.py /home/bitcoin/electrum-personal-server/config.ini
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

5.3 Ativar serviços

sudo systemctl daemon-reload
sudo systemctl enable bitcoind eps
sudo systemctl start bitcoind eps
sudo systemctl status bitcoind eps

Passo 6 — Conectar Sparrow (Whonix Workstation) ao seu EPS .onion

Na Whonix Workstation:

6.1 Configurar servidor no Sparrow

  • Preferences → Server → Private Electrum
  • URL: abcdef123456.onion:50002 (seu .onion)
  • Use Tor: Marcado
  • Use SSL: Não (a menos que tenha configurado certificado no EPS)

6.2 Testar conexão

Clique em Test Connection. Deve ficar verde.

Tempo de conexão inicial: 30-60 segundos (primeira vez que o Tor resolve o .onion).

6.3 Verificar independência total

No Sparrow, vá em View → Console e digite:

getinfo

Deve retornar informações do servidor. Se o server for seu .onion, você está 100% independente.

Passo 8 — Manutenção e monitoramento

8.1 Verificar saúde do EPS

Na VM do node:

sudo journalctl -u eps -f
# Deve mostrar polling e conexões

8.2 Verificar saúde do Bitcoin Core

bitcoin-cli getblockchaininfo
bitcoin-cli getnetworkinfo

8.3 Backup do EPS

O EPS não guarda dados críticos (tudo é derivado da blockchain). Mas faça backup do config.ini (contém sua xpub).

8.4 Atualizações

  • Bitcoin Core: A cada release, baixar, verificar PGP, substituir binários
  • EPS: git pull no repositório, revisar changelog
  • Whonix: sudo apt update && sudo apt full-upgrade na Workstation

Resumo do ecossistema autônomo

Você não depende mais de:

  • Servidores Electrum públicos (que podem logar seus endereços)
  • Coordenadores Whirlpool (só o coordinator, que é inevitável, mas seu node não vaza metadados)
  • Servidores de swap (eigenwallet é P2P, você conecta direto ao maker via Tor)
  • Qualquer serviço clearnet

Seu ambiente está totalmente soberano, anônimo e resiliente.
