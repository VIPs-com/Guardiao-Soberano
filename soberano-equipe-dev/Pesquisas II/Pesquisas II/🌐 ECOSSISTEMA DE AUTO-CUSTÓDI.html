1. Visão Geral da Arquitetura

Princípios Fundamentais

Objetivo Final

Permitir transações diárias de criptomoedas com privacidade máxima, sem expor seed a nenhum sistema conectado à rede, sem depender de servidores públicos de terceiros, e com capacidade de recuperação total em caso de perda de qualquer componente.

4.1 Fluxo Principal: Depósito → Mixagem → Swap → Uso

INÍCIO
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 0: GERAÇÃO DE SEED (Uma vez na vida)                   │
│                                                             │
│ Coldcard + Dice Rolls (99+) → 24 palavras → Placa de aço    │
│ → Passphrase (25ª) → Validar Ian Coleman OFFLINE            │
│ → Testar restore → Exportar xpub                            │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 1: RECEBIMENTO DE BTC                                  │
│                                                             │
│ Adquirir BTC (P2P/RoboSats/DEX)                             │
│ → Enviar para endereço da carteira Sparrow                  │
│ → UTXO aparece na aba UTXOs (Whonix Workstation)            │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 2: COINJOIN (Whirlpool 24/7 no Whonix)                 │
│                                                             │
│ Selecionar UTXOs → Premix → Assinar PSBT (Coldcard)         │
│ → Transmitir → Aguardar confirmação                         │
│ → Iniciar Whirlpool → Aguardar N rounds (mínimo 5)          │
│ → Outputs com ícone azul (anonset ≥ 5)                      │
│ → Mover para Postmix_Whonix                                 │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 3: ISOLAMENTO PARA SWAP                                │
│                                                             │
│ Na carteira Postmix_Whonix:                                 │
│ → Selecionar 1 UTXO (anonset ≥ 5)                           │
│ → Enviar para Swap_Ready_Whonix (mesma xpub, carteira sep.) │
│ → UTXO isolado e rotulado                                   │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 4: SWAP BTC→XMR (Tails ou Whonix)                      │
│                                                             │
│ [Ambiente] Iniciar Tails (sessão limpa) OU Whonix           │
│ [Feather] Gerar subendereço XMR virgem                       │
│ [Sparrow] Gerar endereço BTC change virgem                   │
│ [eigenwallet] Escolher maker → Inserir endereços            │
│ [Coldcard] Assinar PSBT do swap                             │
│ [Aguardar] 25-90 min para conclusão                         │
│ [Feather] Confirmar recebimento XMR                         │
│ [KeePassXC] Registrar TXIDs, endereços, timestamp           │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 5: INTERVALO DE SEGURANÇA                              │
│                                                             │
│ Aguardar MÍNIMO 24h (ideal 3-7 dias)                        │
│ NÃO mover o XMR durante este período                        │
│ Fechar Tails completamente                                  │
│ Whonix continua mixando outros UTXOs                        │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 6: CHURN MONERO (Opcional, recomendado)                │
│                                                             │
│ Feather → Nova carteira (seed virgem)                       │
│ → Enviar XMR para nova carteira                             │
│ → Aguardar 10 confirmações Monero                           │
│ → XMR agora sem vínculo temporal com o swap                 │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ FASE 7: USO DOS FUNDOS                                      │
│                                                             │
│ XMR: Gastar diretamente (Feather)                           │
│ ou                                                        │
│ Novo swap XMR→BTC (via eigenwallet, mesmo processo)         │
│ → BTC pós-swap volta limpo (sem histórico KYC)              │
└─────────────────────────────────────────────────────────────┘

4.2 Fluxo de Recuperação de Desastre

DESASTRE: Whonix Workstation corrompida / Persistent Storage perdido
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ AVALIAR O QUE FOI PERDIDO                                   │
│                                                             │
│ Seed BIP39: ❌ NUNCA perdida (aço + memória)                │
│ Passphrase: ❌ NUNCA perdida (memória + local físico)       │
│ Coldcard: ⚠️ Se perdido, comprar novo e restaurar seed     │
│ VM Whonix Workstation: ⚠️ Restaurar snapshot/backup        │
│ Estado Whirlpool: ⚠️ Restore from Seed no Sparrow          │
│ KeePassXC: ⚠️ Restaurar de backup externo                  │
│ Pendrive Tails: ⚠️ Recriar do zero (sem perda de fundos)   │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│ PROCEDIMENTO DE RECUPERAÇÃO                                 │
│                                                             │
│ 1. Restaurar VM Whonix de backup (snapshot/arquivo .vdi)    │
│    OU recriar VM do zero + restaurar KeePassXC              │
│                                                             │
│ 2. No Sparrow: File → New Wallet → Import xpub do Coldcard  │
│    → Aba Whirlpool → Restore from Seed                      │
│    → Aguardar scan (10-40 min)                              │
│                                                             │
│ 3. Verificar UTXOs na aba Transactions                      │
│    → Outputs de pool reaparecem                             │
│    → Continuar mixagem ou mover para Postmix                │
│                                                             │
│ 4. Recriar pendrive Tails (se perdido):                     │
│    → Baixar ISO, verificar PGP, gravar                      │
│    → Reinstalar eigenwallet, Feather                        │
│    → NÃO há perda de fundos (XMR está na blockchain)        │
└─────────────────────────────────────────────────────────────┘

9. Protocolo de Swap Atômico

9.1 Diagrama do Swap BTC↔XMR (eigenwallet)

┌─────────────────────────────────────────────────────────────┐
│           SWAP ATÔMICO BTC→XMR — SEQUÊNCIA                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  VOCÊ (Taker)                    MAKER (Provedor)           │
│  ───────────                     ─────────────────          │
│                                                             │
│  1. Escolhe maker                                         │
│     → Preço, reputação, bond                               │
│                                                             │
│  2. Fornece endereços:                                    │
│     • BTC change (Sparrow)                                │
│     • XMR destino (Feather)                               │
│                                                             │
│  3. Cria transação BTC            ←──→  4. Maker deposita  │
│     (PSBT para Coldcard)                   bond (caução)    │
│                                                             │
│  5. Assina PSBT (Coldcard)                                │
│     → Transmite transação                                  │
│                                                             │
│  6. Aguarda confirmações          ←──→  7. Maker revela    │

⚠️ Advertências Finais

  1. Nunca digite sua seed em nenhum dispositivo eletrônico conectado à rede.
    O Coldcard existe para isso.
  2. Nunca pule a verificação PGP.
    Um AppImage falso pode roubar todos os fundos.
  3. Nunca reutilize endereços.
    Cada transação merece um endereço virgem.
  4. Nunca consolide UTXOs pós-coinjoin com UTXOs não-misturados.
    Isso desfaz todo o anonimato conquistado.
  5. Nunca mova XMR imediatamente após um swap.
    Aguarde no mínimo 24 horas.
  6. Mantenha backups físicos em locais diferentes.
    Incêndio, inundação ou roubo não podem destruir todos ao mesmo tempo.
  7. Verifique o coordinator Whirlpool a cada nova sessão.
    Coordenadores podem mudar sem aviso.
  8. Teste a recuperação periodicamente.
    Recuperar de mentira uma vez por semestre salva da recuperação real no desespero.
