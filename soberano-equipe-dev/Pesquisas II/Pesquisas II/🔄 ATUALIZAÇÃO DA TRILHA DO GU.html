🔒 NÍVEL 1: O COFRE

Mudanças: ✨ Complemento (Alternativas ao Coldcard)

Passo 1.0 (NOVO) — Escolher seu dispositivo

□ Antes de comprar, avaliar opções:

  ⭐ Coldcard MK5 (Recomendado)
  - Prós: Padrão ouro, SD card, PSBT, Trick PIN, duress wallet
  - Contras: Preço (~$150), só Bitcoin
  - Ideal para: Quem quer o melhor para BTC

  💰 Blockstream Jade (Econômico)
  - Prós: Preço (~$65), QR nativo, open source
  - Contras: Menos recursos que Coldcard
  - Ideal para: Orçamento limitado

  📱 Foundation Passport Batch 2 (QR)
  - Prós: QR nativo, UX amigável, open source
  - Contras: Preço similar ao Coldcard
  - Ideal para: Quem prefere QR a SD card

  🔧 SeedSigner (DIY)
  - Prós: ~$50, stateless, você monta, 100% open source
  - Contras: Requer montagem, aparência frágil
  - Ideal para: Quer construir o próprio dispositivo

  🔧 Krux (DIY Firmware)
  - Prós: Stateless, Taproot, builds reproduzíveis, K210
  - Contras: Hardware específico (Maix Amigo, M5StickV)
  - Ideal para: Alternativa moderna ao SeedSigner

Passo 1.3 — Adendo: Firmware alternativo

  - Flashear firmware no Maix Amigo/M5StickV
  - Verificar build reproduzível (hash publicado)

👁️ NÍVEL 2: A CARTEIRA FRIA

Mudanças: 🔧 Ajuste técnico

Passo 2.1 — Adendo: scurl-download

□ No Tails, ao baixar ferramentas:
  - scurl-download é um wrapper seguro que FORÇA Tor
  - Substitui wget/curl para garantir que nada vaze
  - Uso: scurl-download https://exemplo.com/arquivo
  - Se não disponível: wget sobre Tor já funciona
  - scurl é mais explícito e seguro por padrão

Passo 2.6 — Adendo: QR vs SD Card

□ Além do SD card, testar fluxo via QR:
  - Sparrow exibe PSBT como QR animado (UR2.0)
  - Coldcard/SeedSigner/Krux escaneiam com câmera
  - Dispositivo assina e exibe QR de volta
  - Sparrow escaneia com webcam
  - Útil quando não tem leitor de SD card

🏰 NÍVEL 3: O OBSERVADOR

Mudanças: ✨ Complemento + ➕ Novo Passo

Passo 3.2-A (NOVO) — Stream Isolation

□ Entender o conceito:
  - Cada aplicação usa um circuito Tor SEPARADO
  - Impede correlação entre identidades de carteiras
  - Electrum: --oneserver --server .onion (já isolado)
  - Sparrow: Tor integrado (automático)
  - Verificar no Whonix: uwt (Usability Wrapper for Tor)

Passo 3.2-B (NOVO) — Alternativa: KVM

□ Se seu host é Linux, considere KVM ao invés de VirtualBox
  - KVM é hypervisor nativo do Linux (mais performance)
  - Whonix fornece imagens para KVM
  - Melhor desempenho para node Bitcoin
  - VirtualBox é mais simples para iniciantes
  - Escolha: VirtualBox (começar) → KVM (avançar)

Passo 3.8 (NOVO) — Advertência Crítica

□ ENTENDA O RISCO DO HOST:
  - Whonix protege DENTRO das VMs
  - Se o HOST estiver com keylogger/malware:
    → Tudo que você digita na VM pode ser capturado
    → Senhas, endereços, metadados
  - Mitigações:
    → Host Linux limpo e atualizado
    → Não usar mesmo host para navegação pessoal
    → Qubes OS resolve isso (Nível 7)
    → Coldcard assina OFFLINE (chaves nunca no host)

🔴 CORREÇÃO IMPORTANTE: Passo 5.8

❌ ANTES (incorreto):
"Fazer swap reverso XMR→BTC via eigenwallet"

✅ AGORA (corrigido):
"Fazer swap reverso XMR→BTC via RetoSwap ou BasicSwap"

MOTIVO: eigenwallet (2026) tem XMR→BTC como taker
ainda em desenvolvimento. Para sair do XMR para BTC:

Opção A: RetoSwap (Haveno)
  - Interface P2P, ofertas XMR→BTC disponíveis
  - Escrow multisig, reputação de traders
  - Tempo: horas a dias (depende de contraparte)

Opção B: BasicSwap DEX (avançado)
  - Atomic swap trustless total
  - Requer Docker + nós completos (~400GB)
  - Sem taxas de maker/taker (só mineração)
  - Tempo: varia conforme liquidez

□ Baixar RetoSwap (interface Haveno)
□ Verificar assinatura PGP
□ Instalar no Whonix (persistente)
□ Criar conta (avatar, sem email)
□ Familiarizar-se com a interface
□ Procurar ofertas XMR→BTC para swap reverso

Passo 5.2-B (NOVO) — Instalar BasicSwap (opcional, avançado)

□ Requer Docker + ~400GB (Bitcoin Core + Monero)
□ git clone BasicSwap
□ Seguir guia de instalação
□ Ideal para: trustlessness máxima
□ Contra: complexidade e espaço em disco
□ Só instalar se realmente precisar
