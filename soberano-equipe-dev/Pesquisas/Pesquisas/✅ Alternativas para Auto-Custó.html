✅ Alternativas para Auto-Custódia sem Hardware Wallet (Trezor, Ledger, Coldcard etc.)

Para novatos usando Sparrow no Tails, é possível manter boa segurança com software-only + OPSEC forte, mas o risco é maior que com hardware wallet (chaves ficam mais expostas a malware, keyloggers ou erros humanos). A comunidade recomenda: comece com valores muito pequenos (ex: < R$ 2.000) e evolua para hardware depois.

Aqui estão as principais alternativas usadas em 2026, ordenadas do mais simples ao mais seguro para novatos:

1. Software Wallet com Air-Gapped Signing (Recomendado para Novatos sem Hardware)

  • Como funciona: Gere a seed em um computador air-gapped (sem internet, nunca conectado), use Sparrow no Tails apenas como watch-only (só vê saldo e cria transações não assinadas – PSBT). Assine offline e transmita depois.
  • Passos básicos no Tails/Sparrow:
    • Crie wallet Software no Sparrow (offline).
    • Exporte xpub/zpub para watch-only no Tails online.
    • Para assinatura: Use um segundo USB/Tails boot air-gapped (sem rede) para assinar PSBT.
  • Vantagens: Grátis, usa o que você já tem.
  • Desvantagens: Mais complicado que hardware; risco se o air-gapped for contaminado.
  • Popular na comunidade: Sparrow + Electrum em modo offline/air-gapped.

2. Paper Wallet ou Metal Backup (Clássico e Barato)

  • Gere seed BIP39 offline (use dados de entropia boa: dados de dice rolls ou /dev/random no Tails).
  • Crie endereços recebíveis no Sparrow (watch-only).
  • Guarde a seed em papel laminado ou melhor: placa de metal (ex: Billfodl, Cryptosteel ou DIY com placa de aço).
  • Nunca digite a seed em máquina online. Use só para restore quando necessário.
  • Sparrow: Importe como watch-only e use PSBT para gastar.
  • Status em 2026: Ainda viável para holdings pequenas, mas considerado obsoleto por muitos devido a riscos de perda/fogo/água e erros de digitação no restore. Menos prático que hardware.

3. Multisig com Múltiplos Dispositivos/Seeds (Boa Opção Intermediária)

  • Crie uma wallet 2-of-3 ou 2-of-2 no Sparrow:
    • Uma seed no Tails (software).
    • Outra seed em papel/metal ou em outro computador air-gapped.
    • Terceira opcional em mobile (GrapheneOS).
  • Sparrow tem suporte excelente a multisig + PSBT.
  • Vantagens: Tolerância a perda de uma seed/dispositivo. Mais seguro que single sig software.
  • Popular: Muitos usam Sparrow para multisig sem hardware no início.

4. Mobile como Semi-Cold (GrapheneOS/CalyxOS)

  • Use o celular dedicado (Pixel com GrapheneOS) como wallet principal com seed gerada offline.
  • Apps: BlueWallet, Electrum ou Sparrow Mobile (watch-only).
  • Mantenha o celular sem internet (modo avião + Orbot só quando necessário) para operações sensíveis.
  • Limitação: Mobile é menos seguro que desktop air-gapped (risco de apps maliciosos, perda física).

5. Outras Ferramentas/Abordagens Avançadas (Menos para Novatos)

  • SeedSigner ou Krux: Projetos DIY open-source (roda em Raspberry Pi barato ou dispositivo dedicado) para signing air-gapped via QR code. Bom substituto barato de hardware.
  • AirGap Vault (app mobile): Gera seed offline no celular e usa QR codes com Sparrow.
  • Zengo (MPC seedless): Elimina seed tradicional, mas é mais custodial em alguns aspectos (não puramente self-custody como Sparrow).
  • Bitcoin Core + Sparrow: Para node completo e máxima privacidade, mas mais pesado.

Recomendação para Seu Caso (Novato + Sparrow no Tails)

  1. Comece com Air-Gapped Software + Watch-Only no Tails (mais alinhado ao que você já está montando).
  2. Use multisig 2-of-3 com seeds em papel/metal + Tails.
  3. Backup forte: Seed em metal + teste de restore em ambiente controlado.
  4. OPSEC essencial:
    • Sempre verifique assinaturas.
    • Nunca conecte seed online.
    • Use Tails amnésico para tudo.
    • Transações pequenas primeiro.

Hardware continua sendo o padrão ouro da comunidade para holdings significativas (menos risco humano). Mas essas alternativas permitem começar hoje sem gastar.
