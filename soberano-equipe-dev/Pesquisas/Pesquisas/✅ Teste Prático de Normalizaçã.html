✅ Teste Prático de Normalização Unicode (NFKD) no Sparrow Wallet

Sparrow segue estritamente o padrão BIP39, que exige NFKD na passphrase. Aqui está como testar isso de forma segura no seu Tails.

Método de Teste (Use Testnet!)

  1. Prepare o Sparrow
    • Abra o Sparrow no Tails.
    • Vá em File → Preferences → marque Testnet.
    • Crie uma nova wallet Single Signature com uma seed simples (gere nova).
  2. Crie uma Passphrase com Caracteres Problemáticos
    • Feche e reabra a wallet.
    • Quando o Sparrow pedir a BIP39 Passphrase, digite algo com acentos:
      • Exemplo: cafébitcoin2026
      • Ou: João2026!Soberania
  3. Observe o Comportamento
    • Gere alguns endereços de Receive.
    • Anote 1-2 endereços.
    • Feche completamente o Sparrow (feche o programa).

  4. Reabra e Teste Variações

    • Reabra a mesma wallet.
    • Tente diferentes formas da passphrase:
      • cafébitcoin2026 (com acento)
      • cafebitcoin2026 (sem acento)
      • cafébitcoin2026 (e + acento separado, se conseguir digitar)

    Resultado esperado:

    • Sparrow aplica NFKD automaticamente.
    • Apenas a forma correta (depois da normalização NFKD) vai mostrar os endereços e saldo de teste.
    • Qualquer variação errada gera uma wallet totalmente diferente (endereços diferentes, saldo zero).
  5. Teste com Passphrase ASCII (Recomendado)
    • Repita o teste com:
      • cafebitcoin2026 (só ASCII)
      • BitcoinSoberania2026!
    • Você verá que funciona de forma consistente.

O que o Teste Ensina

  • Sparrow + SeedSigner/Krux aplicam NFKD da mesma forma (compatibilidade).
  • Usar acentos ou caracteres especiais pode gerar ambiguidade se não for consistente.
  • Por isso a recomendação forte: use apenas ASCII na passphrase.

Dica de Segurança: Sempre teste restore com a passphrase exata antes de usar com fundos reais.
