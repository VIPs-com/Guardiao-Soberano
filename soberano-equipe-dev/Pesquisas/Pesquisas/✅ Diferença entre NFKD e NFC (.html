✅ Diferença entre NFKD e NFC (Normalização Unicode)

Aqui está uma explicação clara e prática, especialmente no contexto de BIP39 Passphrase:

1. O que é Normalização Unicode?

Quando você digita caracteres com acento (á, é, ç, ã, õ, ü, etc.), o computador pode representá-los de formas diferentes internamente. A normalização padroniza isso para evitar incompatibilidades.

2. NFC vs NFKD — Comparação Direta

Exemplos Concretos

  • Palavra: "João"
    • NFC: João (mantém o ã como um caractere)
    • NFKD: João (o til é separado do "a")
  • Palavra: "Straße" (alemão)
    • NFC: Straße
    • NFKD: Strasse (ß vira ss)
  • Passphrase: CaféBitcoin2026
    • Com NFC: Pode gerar uma wallet
    • Com NFKD: Gera uma wallet diferente

Por que o BIP39 exige NFKD?

O padrão BIP39 escolheu NFKD para garantir que qualquer implementação (SeedSigner, Krux, Sparrow, Electrum, etc.) gere exatamente a mesma wallet a partir da mesma seed + passphrase.

Se um dispositivo usasse NFC e outro NFKD, você teria duas wallets diferentes com a mesma frase visual.

Recomendação Forte para Você

  • Sempre use apenas caracteres ASCII na passphrase:
    • Letras A-Z, a-z
    • Números 0-9
    • Símbolos: ! @ # $ % * ( ) - _ + =

  • Exemplo seguro:

    BitcoinSoberaniaForte2026!
  • Evite completamente acentos, ç, ã, õ, emojis, etc. Isso elimina qualquer risco de normalização.

Resumo simples:

  • NFC = Forma "bonita" e composta (mais comum no dia a dia).
  • NFKD = Forma "desmontada" e compatível (usada em criptografia para consistência).

No seu setup com SeedSigner/Krux + Sparrow, os dispositivos já aplicam NFKD automaticamente. Você só precisa evitar caracteres problemáticos.
