✅ Exploração Técnica da Especificação BIP39 (Mnemonic Code for Generating Deterministic Keys)

A BIP39 é uma das Bitcoin Improvement Proposals mais importantes. Ela define como gerar uma frase mnemônica (seed phrase) fácil de lembrar a partir de entropia aleatória, e como converter essa frase em uma seed binária usada para derivar chaves (via BIP32, BIP44, etc.).

1. Objetivo Principal

  • Transformar uma sequência aleatória (entropia) em palavras humanas (mnemônica) para backup e recuperação de wallets.
  • Adicionar suporte opcional a passphrase (Word 25) para proteção extra.
  • Gerar uma seed determinística de 512 bits.

2. Geração da Mnemônica (Mnemonic)

Passos técnicos (conforme a spec):

  1. Gere entropia inicial (ENT) em múltiplos de 32 bits:
    • 128 bits → 12 palavras
    • 160 bits → 15 palavras
    • 192 bits → 18 palavras
    • 224 bits → 21 palavras
    • 256 bits → 24 palavras (mais comum e recomendado)
  2. Calcule o checksum:
    • Pegue o SHA256 da entropia.
    • Use os primeiros ENT / 32 bits como checksum.
    • Exemplo: 128 bits de entropia → 4 bits de checksum.
  3. Concatene entropia + checksum.
  4. Divida em grupos de 11 bits (cada grupo = um número de 0 a 2047).
  5. Converta cada número em uma palavra da wordlist (2048 palavras).

3. Wordlist

  • Lista com exatamente 2048 palavras (2¹¹).
  • Ordenada alfabeticamente.
  • Primeiras 4 letras são únicas (facilita digitação).
  • Evita palavras semelhantes (build/built, woman/women etc.).
  • A oficial é em inglês (fortemente recomendada). Outras línguas existem, mas são desencorajadas para compatibilidade.

As palavras devem ser codificadas em UTF-8 NFKD.

4. Conversão da Mnemônica para Seed (PBKDF2)

Esta é a parte mais importante tecnicamente:

  • Password: A frase mnemônica (palavras separadas por espaço) em UTF-8 NFKD.
  • Salt: "mnemonic" + passphrase (também em UTF-8 NFKD). Se não houver passphrase, usa string vazia.
  • Função: PBKDF2 com:
    • HMAC-SHA512
    • 2048 iterações
    • Comprimento de saída: 512 bits (64 bytes)

Essa seed de 512 bits é então usada no BIP32 (HD wallets) para derivar todas as chaves.

5. Passphrase (BIP39 Word 25)

  • Totalmente opcional.
  • Gera uma wallet diferente da mesma seed.
  • Aplica NFKD (como explicado antes).
  • Fornece plausible deniability (toda passphrase gera uma wallet válida, mas só a correta tem os fundos).

6. Pontos Técnicos Importantes

  • A mnemônica não é uma brainwallet (não deve ser criada manualmente pelo usuário).
  • O checksum permite detectar erros de digitação.
  • Implementações devem validar o checksum.
  • A spec desencoraja strongly o uso de wordlists não-inglesas em produção.
