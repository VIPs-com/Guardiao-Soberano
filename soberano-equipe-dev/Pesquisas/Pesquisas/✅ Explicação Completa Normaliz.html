✅ Explicação Completa: Normalização NFKD no BIP39 (Passphrase)

A NFKD (Normalization Form Compatibility Decomposition) é uma regra técnica importante do padrão BIP39 para garantir que a passphrase funcione de forma idêntica em todos os wallets (Sparrow, SeedSigner, Krux, Electrum, etc.).

O que é NFKD?

É um processo de normalização Unicode que transforma caracteres especiais de uma forma padronizada antes de usar a passphrase para derivar a seed.

Exemplos práticos:

Por que isso importa no BIP39?

  • A passphrase é usada junto com a seed para derivar as chaves privadas.
  • Se dois wallets aplicarem a normalização de forma diferente, eles geram wallets distintas (mesma seed + mesma passphrase visual = fundos diferentes!).
  • O padrão BIP39 exige que todos os wallets usem NFKD na passphrase.

Como os Dispositivos Lidam com Isso

  • SeedSigner: Aplica NFKD automaticamente (boa implementação).
  • Krux: Também aplica NFKD e tem avisos recentes sobre o tema.
  • Sparrow Wallet: Aplica corretamente quando você importa via xpub (o xpub já carrega o resultado normalizado).
  • Outros wallets (menos confiáveis): Alguns apps mobile ou sites antigos podem não seguir o padrão corretamente.

Recomendações Práticas para Você (Novato)

  1. Use apenas caracteres ASCII na passphrase (mais seguro e simples):

    • Letras maiúsculas/minúsculas (A-Z, a-z)
    • Números (0-9)
    • Símbolos comuns: ! @ # $ % & * ( ) - _ = +

    Exemplo bom:

    BitcoinSoberania2026!Segura
  2. Evite:
    • Acentos (á, é, ç, ã, õ)
    • Caracteres especiais Unicode (emoji, letras de outros alfabetos)
    • Espaços no início/fim
  3. Se precisar usar acentos:
    • Teste sempre com small amount.
    • Use a mesma ferramenta/dispositivo para gerar e restaurar.

Resumo Importante

  • NFKD = Regra de compatibilidade para que “a mesma passphrase” produza o mesmo resultado em qualquer wallet compatível com BIP39.
  • Melhor prática: Mantenha a passphrase em ASCII puro → zero dor de cabeça com normalização.
  • Sempre que você digitar a passphrase no SeedSigner/Krux, o dispositivo já faz a normalização NFKD internamente.
