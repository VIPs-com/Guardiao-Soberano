✅ Tutorial Completo: Sparrow Wallet no Tails OS (com Persistência – 2026)

Este é um guia prático e seguro para novatos no nível que estamos trabalhando. O objetivo é ter o Sparrow persistente no volume criptografado do Tails, sem precisar reinstalar toda vez.

⚠️ Avisos Importantes Antes de Começar

  • Nunca armazene seeds ou chaves privadas diretamente no Sparrow no Tails com persistência (use como Watch-Only + hardware wallet).
  • Sempre verifique as assinaturas e hashes dos downloads.
  • Use quantias pequenas no início para testar.
  • Tails é amnésico: sem persistência, tudo desaparece ao desligar.

Passo 1: Prepare o Tails com Persistência

  1. Boot no Tails (USB live).
  2. Vá em Applications → Tails → Persistent Storage (ou Configure Persistent Volume).
  3. Crie o volume persistente com uma senha forte (5-7 palavras aleatórias).
  4. Ative as seguintes opções:
    • Persistent Folder (Personal Documents)
    • Dotfiles (Advanced Settings)
    • Opcional: Network Connections e Welcome Screen
  5. Reinicie o Tails e desbloqueie o Persistent Storage ao boot.

O diretório ~/Persistent aparecerá no File Browser.

Passo 2: Baixe e Verifique o Sparrow (Muito Importante!)

  1. Abra o Tor Browser no Tails.
  2. Acesse: https://sparrowwallet.com/download/
  3. Baixe a versão Linux (Intel/AMD) Standalone (sparrowwallet-2.x.x-x86_64.tar.gz).
  4. Baixe também:
    • sparrow-2.x.x-manifest.txt
    • sparrow-2.x.x-manifest.txt.asc

Verificação (recomendado):

  • No Tails, abra Terminal.

  • Importe a chave (se necessário):

    curl https://keybase.io/craigraw/pgp_keys.asc | gpg --import

  • Verifique assinatura:

    gpg --verify sparrow-*.manifest.txt.asc

    (Deve mostrar “Good signature”)

  • Verifique hash:

    sha256sum --check sparrow-*-manifest.txt --ignore-missing

    (Deve mostrar “OK”)

Passo 3: Organize e Instale no Persistent Storage

No Terminal:

# Crie pastas
mkdir -p ~/Persistent/Programas/Sparrow/data

# Extraia o Sparrow (substitua pelo nome exato do arquivo baixado)
tar -xvzf ~/Downloads/sparrowwallet-*-x86_64.tar.gz -C ~/Persistent/Programas/Sparrow --strip-components=1

Teste se funciona:

~/Persistent/Programas/Sparrow/bin/Sparrow -d ~/Persistent/Programas/Sparrow/data

Feche o Sparrow após o teste.

Passo 4: Crie Atalho no Menu de Aplicativos

mkdir -p /live/persistence/TailsData_unlocked/dotfiles/.local/share/applications
nano /live/persistence/TailsData_unlocked/dotfiles/.local/share/applications/Sparrow.desktop

Cole o seguinte conteúdo no nano (Ctrl+O para salvar, Ctrl+X para sair):

[Desktop Entry]
Name=Sparrow
Comment=Sparrow Bitcoin Wallet
Exec=/home/amnesia/Persistent/Programas/Sparrow/bin/Sparrow -d /home/amnesia/Persistent/Programas/Sparrow/data %U
Icon=/home/amnesia/Persistent/Programas/Sparrow/lib/Sparrow.png
Terminal=false
Type=Application
Categories=Unknown
MimeType=application/psbt;application/bitcoin-transaction;x-scheme-handler/bitcoin;x-scheme-handler/auth47;x-scheme-handler/lightning

Copie o atalho para o diretório atual (para testar sem reboot):

cp /live/persistence/TailsData_unlocked/dotfiles/.local/share/applications/Sparrow.desktop ~/.local/share/applications/

Passo 5: Configure o Sparrow

  1. Abra o Sparrow pelo menu (ou terminal).
  2. No primeiro boot, configure o servidor:
    • Vá em File → Preferences → Server
    • Marque Use proxy
    • Proxy URL: 127.0.0.1 ou localhost
    • Port: 9050 (Tor do Tails)
  3. Recomendado: Use um servidor Electrum próprio ou público via Tor para mais privacidade.
  4. Crie uma wallet Watch-Only (importe xpub/zpub do hardware wallet).

Atualização Futura do Sparrow

  1. Baixe a nova versão Standalone + verifique.
  2. Delete as pastas bin e lib antigas em ~/Persistent/Programas/Sparrow/.
  3. Extraia a nova versão no lugar.
  4. Mantenha a pasta data intacta (ela tem suas wallets e configs).

Dicas Extras de Segurança

  • Use sempre com hardware wallet (Trezor/Coldcard) para assinatura offline (PSBT).
  • Para hardware: vá em Tools → Install udev rules (execute o comando mostrado).
  • Faça backup do volume persistente periodicamente.
  • Teste tudo com valores pequenos primeiro.
  • Combine com node próprio (mais avançado).
