✅ Teste de Restore com Passphrase no Sparrow Wallet (Tails + Air-Gapped)

Este é um teste obrigatório antes de usar com valores reais. Vamos usar Testnet para zerar o risco.

Preparação (Faça Isso Primeiro)

  1. Boot no Tails com persistência desbloqueada.
  2. Abra o Sparrow.
  3. Vá em File → Preferences:
    • Marque Testnet (ou Signet).
    • Configure servidor via Tor (localhost:9050).
  4. Tenha sua seed + passphrase anotadas em papel/metal.
  5. Tenha o Output Descriptor salvo (do Sparrow original: Settings → Output Descriptor → Copy).

Método 1: Restore via xpub do Signer (Mais Recomendado para Seu Setup)

  1. No SeedSigner ou Krux (air-gapped):
    • Carregue a seed de 12/24 palavras.
    • Aplique exatamente a mesma passphrase (BIP39 Word 25).
    • Exporte o xpub/zpub via QR (Native Segwit).
  2. No Sparrow:
    • File → New Wallet.
    • Dê um nome (ex: “TesteRestorePassphrase”).
    • Escolha Single Signature ou Multi Signature (dependendo do seu caso).
    • Adicione o Keystore:
      • Air Gapped Hardware Wallet → SeedSigner ou Krux.
      • Scan QR do xpub (com passphrase já aplicada no signer).
    • Clique Apply / Create.
  3. Verifique:
    • Vá na aba Receive → gere um endereço.
    • Compare com o endereço original (se tiver).

Método 2: Restore via Output Descriptor (Mais Rápido)

  1. No Sparrow:
    • File → New Wallet.
    • Nome → Continue.
    • Em vez de adicionar keystores manualmente, clique em Edit ao lado de Output Descriptor.
    • Cole o descriptor completo que você salvou anteriormente.
    • Clique OK.
  2. Sparrow vai carregar a wallet com todos os keystores.
  3. Se for uma wallet com passphrase, o descriptor já inclui o efeito dela (não precisa digitar novamente).

Passo Final: Teste Prático com Fundos de Testnet

  1. Gere um endereço na wallet restaurada.
  2. Consiga Testnet BTC grátis (faucets como testnet-faucet.mempool.co ou outros).
  3. Envie para o endereço.
  4. Aguarde confirmações (use Blockstream Testnet Explorer).
  5. Verifique o saldo no Sparrow.
  6. Crie uma transação de saída (small amount) usando PSBT + QR com o signer.
  7. Broadcast e confirme que funciona.

Problemas Comuns e Soluções

  • Saldo não aparece → Quase sempre passphrase errada. Feche a wallet, reabra e digite a passphrase com cuidado (Sparrow não guarda a passphrase derivada por segurança).
  • Endereços diferentes → Passphrase incorreta ou derivation path errado (m/84'/0'/0' para Native Segwit).
  • Erro ao importar → Verifique se o signer está com a passphrase aplicada no momento da exportação do xpub.

Após o teste em Testnet:

  • Delete a wallet de teste.
  • Repita o processo em Mainnet só com small amount real.
  • Faça backup do novo Output Descriptor.

Dica de Ouro: Sempre teste restore completo (seed + passphrase) antes de colocar valores significativos.
